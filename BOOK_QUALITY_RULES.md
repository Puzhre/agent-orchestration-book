# Book Quality Rules v1

> 核心理念：精炼干货，洞见驱动。每段内容必须有insight，无insight不入书。废话=毒药。

## 章节质量评分体系（百分制）

| 维度 | 权重 | 1分(垃圾) | 5分(及格) | 10分(出版级) |
|------|------|-----------|-----------|-------------|
| 深度 | 30% | 表面列举what | 有分析有对比 | 透视本质，揭示why和how不只what |
| 实证 | 25% | 无代码/无引用 | 有代码片段 | 真实项目源码对比+实测数据+可复现 |
| 结构 | 20% | 散文堆砌 | 有逻辑链 | 金字塔原理，每节回答一个核心问题 |
| 新知 | 15% | 重复文档搬运 | 有独到见解 | 反直觉洞察，读过源码也未必知道 |
| 可操作 | 10% | 理论空谈 | 有步骤指引 | 可直接copy-paste跑的代码+配置 |

## 毕业门槛（章节通过审核的硬条件）

- **加权总分 >= 92**
- **单维度 >= 8**（任一维度低于8则整章打回）
- **实证维度 >= 9**（实证是最硬的硬指标，低于9不行）
- **中英双语完整**（英文为主，中文翻译不降质）

**未毕业章节标记为draft，不出现在正式目录。**

## 内容准入6关（素材/新内容进入书之前，全部通过才可写入）

1. **不是文档搬运** — 不能把README/文档翻译一遍就塞进来，必须有提炼、重组和升华
2. **交叉验证** — 至少2个独立项目/论文佐证同一观点，单一来源不采信
3. **读者视角** — 一个没读过源码的人读完这节能理解并应用吗？说不清就重写
4. **增量价值** — 这段内容网上搜不到吗？搜得到就必须比现有资料深一层
5. **可追溯** — 每个引用必须链接到源（仓库URL/论文DOI/博客URL），无来源不入书
6. **洞见锐度** — 这个素材能产出反直觉/非显而易见的insight吗？显而易见的常识不要

## 爬取→策展→写入 流程

```
爬取(cron 4h) → 缓存本地(.crawl-cache/)
     → 策展(hermes agent深度分析，提取insight)
     → 准入审查(6关)
     → 注入章节(只注入通过审查的insight)
     → 章节整体重评(注入后该章重新评分，分数不能下降)
     → commit+push
```

### 策展要求

策展不是总结，是提炼。每个素材必须产出：
1. **核心insight**（1-3句话，反直觉或非显而易见）
2. **支撑证据**（代码片段/配置/数据）
3. **归属章节**（该insight属于哪章哪节）
4. **与已有内容的差异**（和书里已写的有什么不同，增量在哪）

没有insight的素材直接丢弃，不存入缓存。

## 写入规则

### 每段内容必须满足

- 有且仅有一个核心观点
- 有实证支撑（代码/数据/案例）
- 有insight（不是常识重述）
- 精炼，无冗余修饰词

### 禁止写入

- "众所周知""众所周知的是" 这类废话开头
- 没有代码/案例支撑的断言
- 从文档原文大段搬运（>3行连续原文必须改写）
- 凑字数的过渡段（"下面我们来看看""接下来"）
- 没有增量价值的重复表述
- 搜索结果直接贴入（必须经过理解-提炼-重组）

### 代码示例标准

- 必须可运行（不是伪代码）
- 必须有注释解释关键决策
- 必须标注来源项目
- 必须说明适用场景和局限

## 章节模板

每章必须包含：

```markdown
# Chapter N: Title

## N.1 Core Question
本章回答的一个核心问题

## N.2-N.X Main Sections
每个section解决一个子问题

## N.Y Cross-Project Comparison (如适用)
至少2个项目的横向对比

## N.Z Key Insights
本章所有insight的精炼汇总（bullet points，每条<=2句话）

## References
- [1] 来源链接
```

## 评分流程

1. **自评**：hermes agent写完/改完章节后自评打分
2. **交叉审**：另一个hermes实例（default主实例GLM-5.1）按本规则独立评分
3. **争议裁决**：两个评分差距>10分时，标记争议点，由用户最终裁决
4. **重评**：每次修改后重新评分，分数不能低于修改前

## 阶段定义

| 阶段 | 目标 | 准入 | 准出 |
|------|------|------|------|
| 爬取积累 | 收集高质量素材 | 无 | 素材库>=50条带insight的条目 |
| 章节填充 | 把素材写入各章 | 素材通过准入6关 | 所有章节加权>=92 |
| 精炼打磨 | 删除冗余，强化insight | 所有章节>=92 | 每章<=最优长度(无废话可删) |
| 终审 | 整书一致性检查 | 精炼完成 | 跨章引用一致+无重复+术语统一 |

## 禁止事项（铁律）

- 绝不写入没有insight的内容
- 绝不搬运文档原文（改写+提炼+升华）
- 绝不在实证不足时下断言
- 绝不凑字数（精炼>篇幅）
- 绝不跳过准入6关就写入
- 绝不让评分下降（修改后分数必须>=修改前）
- 单维度<8的章节绝不标记为published
